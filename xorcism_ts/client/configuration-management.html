Configuration Management

One governance view over the secure-configuration content library — the OVAL/SCAP checks XORCISM holds. The configuration items are the compliance-class hardening baselines; the broader library also carries patch, vulnerability and inventory checks. Each baseline carries a health score; the worklist surfaces the gaps that matter: deprecated content still present, baselines never verified by a scan, interim-status checks, and hardening checks with no CCE reference. Run scans from the OVAL agent; browse content under OVAL definitions.

⚡ Launch an OVAL scan

Pick a host (localhost or remote) and a scan type — the enrolled XOR agent runs it at its next check-in; results land in the verification cards below. Set a recurrence to schedule it on a cadence.

🛡️ Deploy compliance policies to the fleet

Push a system compliance policy — CIS Benchmark / OVAL hardening baseline — to your enrolled XOR agents in one shot. Each agent evaluates the compliance content at its next check-in (CCE/CCSS checks); pass/fail verdicts land in the verification cards below and in OVAL scan results. Add a recurrence to keep re-evaluating it and catch configuration drift continuously (enforcement = continuous assurance).
